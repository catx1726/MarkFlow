## 2. 核心摘要 (Core Summary)

### 🚀 最终解决问题的关键点

- **引导卡片集成 (Onboarding Card)**:
- **方案**: 在设置页面 (`Options.vue`) 顶部新增一个静态“欢迎使用”卡片。
- **视觉设计**: 采用蓝色左边框 (`border-l-4 border-blue-500`) 增强视觉锚点，确保在不破坏现有简洁 UI 的前提下，实现对新用户的有效引导。
- **内容整合**: 模块化展示“快速开始”、“核心功能”与“致谢支持”，一站式解决新用户的上手困惑。

### 🧠 关键技术点 (Knowledge Points)

- **轻量化开发决策**: 坚持 **KISS 原则 (Keep It Simple, Stupid)**。由于引导内容相对固定，选择直接在 Vue 模板中静态嵌入，避免了引入额外的状态管理或动态组件，保持了设置页面的加载效率。
- **样式一致性维护**: 全面复用项目已有的 **UnoCSS/Tailwind CSS** 原子化类。通过 `dark:` 修饰符原生适配深色模式，确保了新 UI 元素在不同主题下的视觉稳定性。
- **防御式 UI 实现**:
- **安全导航**: 对所有外部链接强制执行 `target="_blank"`，防止设置页面被意外跳转丢失。
- **样式隔离**: 遵循 `.setting-card` 规范，杜绝全局样式污染，确保组件的高内聚性。

## 3. 沉淀与反思 (Post-mortem)

### ⚠️ 避坑指南 (Lessons Learned)

- **硬编码的局限性 (Hardcoding Trap)**:
- **风险**: 引导文案中硬编码了 `Alt` 触发键。若用户在下方设置中修改了快捷键，顶部的引导说明将产生歧义。
- **教训**: **涉及用户配置的文案应具备“动态意识”**。未来应通过 `settings.ts` 动态映射配置项，而非编写静态文案。

### 💎 复用价值

- **通知/引导模式**: 本次实现的卡片结构可抽象为通用的 `InfoCard` 组件，适用于后续的版本更新通知或重要功能预告。
